Set up port forwarding for streaming – step by step, for every router

For your friends to see your stream, your router has to let two ports through to your PC. Most of the time Lumora does this automatically. If it doesn't work on your router, here's the exact click-by-click guide – for every common device, including the step most people get stuck on.

Do I even need this page?

Usually no. Lumora opens the ports itself when you start streaming (via UPnP) and automatically uses the IPv6 direct path on connections without a public IPv4 address. You only need this guide if Lumora shows in the Stream tab: “Router isn't opening the ports automatically”.

What needs to be forwarded

  • Port 8787 (TCP) – the connection signalling.
  • Port 8189 (UDP) – the actual video and audio.
  • Both point to the same PC (the one you stream from). External = internal, nothing gets rewritten.

The basic pattern – it works the same on every router

  1. Open the router menu: type the router's address in your browser (usually printed on a sticker on the device) and sign in.
  2. Select your PC as the device: almost every router binds the rule to a device. This is exactly where most people get stuck – the PC has to be selected or added first. If it doesn't show up, give it a static IP first (in the router under “DHCP” / “Address Reservation”), otherwise the rule points nowhere after a reboot.
  3. Forward port 8787 (TCP) and save.
  4. Forward port 8189 (UDP) – on most routers a second, separate rule.

Pick your router

Germany: FRITZ!Box / 1&1 · Telekom Speedport · Vodafone Station · o2 HomeBox
Brands: TP-Link · ASUS · Netgear · D-Link · Linksys · Huawei
International: BT Smart Hub · Sky · Virgin Media · Xfinity · AT&T

FRITZ!Box (also 1&1 HomeServer)

1&1 HomeServers are FRITZ!Box models – same guide.

Open fritz.box in your browser and sign in.

  1. Go to “Internet › Permit Access › Port Sharing”.
  2. Click “Add Device for Sharing” and pick this PC in the “Device” field. If it's missing, choose “Enter IP address manually”.
  3. “New Port Sharing” › “Port Sharing”; under “Application” choose “Other application”. Name it “Lumora TCP”, protocol TCP, port 8787 in every port field. Save with “OK”.
  4. Add another: “Lumora UDP”, protocol UDP, port 8189. Save.
  5. Confirm with “Apply” at the bottom.

Tip: tick “Allow independent port sharing for this device” on the PC entry and Lumora can open the port itself again next time.

Telekom Speedport (Smart 3 / 4)

Open speedport.ip (or 192.168.2.1) and sign in with the device password (on the back of the router). Speedport routers don't support UPnP – manual forwarding is the only way here.

  1. Static IP first: “Network › Connected devices” › open this PC via “show” › tick “Always the same IPv4 address” › Save.
  2. Open “Internet › Port forwarding” and enter a name (e.g. “Lumora Stream”).
  3. Select this PC (the device) from the list.
  4. Without a template, enter port 8787 (public = internal) and save.
  5. Use “Add further port forwarding” to create a second rule with port 8189. Save.

Vodafone Station (cable)

Open 192.168.0.1, click the menu icon at the top right and sign in (password on the sticker).

  1. Open “Internet › Port-Mapping” and click the “Add” icon.
  2. Service name “Lumora TCP”, Device = this PC, LAN IP = its home-network IP, protocol TCP, Type “Enable port”, WAN port 8787, LAN port 8787.
  3. “Save” › “Apply” › drag the slider to the right to activate.
  4. Add a second rule the same way, but protocol UDP and port 8189.

If there's no “Port-Mapping” menu, you have a DS-Lite line – see below.

o2 HomeBox

Open o2.box (or 192.168.1.1) and sign in (password on the back of the device).

  1. Give the PC a static IP (DHCP reservation in the router).
  2. Open the “IPv4/IPv6 port forwarding” menu (supports TCP and UDP).
  3. New rule: name the service freely, Computer IP = this PC, port 8787, protocol TCP; save.
  4. Second rule: port 8189, protocol UDP.

o2 cable often ships FRITZ!Box models – then use the FRITZ!Box guide.

TP-Link (Archer)

Open tplinkwifi.net (or 192.168.0.1) and sign in.

  1. Open “Advanced › NAT Forwarding › Virtual Servers” (on some models “Port Forwarding”).
  2. “Add”: Service Name “Lumora-TCP”, External + Internal Port 8787, protocol TCP; Internal IP = this PC (pick it via “View Connected Devices”). Save.
  3. “Add” again: “Lumora-UDP”, port 8189, protocol UDP, same PC IP. Save.

A fixed IP is recommended (Address Reservation). UPnP alternative: enable “Advanced › NAT Forwarding › UPnP”.

ASUS (ASUSWRT)

Open router.asus.com (or 192.168.1.1) and sign in.

  1. On the left open “WAN” › tab “Virtual Server / Port Forwarding”; set “Enable Port Forwarding” to ON.
  2. “Add profile”: Service Name “Lumora-TCP”, protocol TCP, External + Internal Port 8787, Internal IP = your PC's IP. Apply.
  3. Second “Add profile”: protocol UDP, port 8189, same PC IP. Save with “Apply”.

Give a fixed IP via DHCP reservation. UPnP alternative: “WAN › Internet Connection › Enable UPnP: Yes”.

Netgear (Nighthawk / Orbi)

Open routerlogin.net, user “admin” and your password.

  1. “ADVANCED › Advanced Setup › Port Forwarding/Port Triggering”; keep “Port Forwarding” selected.
  2. “Add Custom Service”: Name “Lumora-TCP”, Type TCP, External Starting + Ending port 8787; Server IP = this PC (selectable in the device list below). “Apply”.
  3. “Add Custom Service” again: Type UDP, port 8189. “Apply”.

Give a reserved IP first. UPnP alternative: “ADVANCED › Advanced Setup › UPnP › Turn UPnP On”.

D-Link

Open dlinkrouter.local (or 192.168.0.1) and sign in.

  1. At the top go to “Features › Port Forwarding”.
  2. “Add Rule”: Name “Lumora”, Local IP = this PC (selectable from the dropdown).
  3. Enter TCP port 8787 and UDP port 8189 in their fields (on D-Link both ports fit in one rule).
  4. Set “Schedule” to “Always Enable”, then “Save”.

Linksys (Smart Wi-Fi)

Open LinksysSmartWiFi.com (or 192.168.1.1); sign in locally via “click here” with the router password.

  1. “Router Settings › Security › Apps and Gaming › Single Port Forwarding”.
  2. “Add a new Single Port Forwarding”: “Lumora TCP”, External + Internal Port 8787, protocol TCP, Device IP# = this PC. Tick “Enabled”.
  3. Second rule: “Lumora UDP”, port 8189, protocol UDP. Tick “Enabled”. Then “Save” › “Apply” › “Ok”.

Linksys requires a fixed IP – give the PC a static/reserved IP first.

Huawei

Open 192.168.3.1 and sign in (password on the label).

  1. Open “More Functions › Security Settings › NAT Services”.
  2. Click the plus/“Add” icon next to “Port Forwarding”.
  3. Service name “Lumora TCP”, Device name = this PC (from the device list – the PC must be connected), protocol TCP, internal + external port 8787. “Save”.
  4. Second rule: “Lumora UDP”, protocol UDP, port 8189. “Save”.

UPnP is off by default: enable “More Functions › Network Settings › UPnP” so Lumora's automatic setup works.

BT Smart Hub

Open 192.168.1.254 (admin password on the hub card / back).

  1. Open “Advanced Settings › Firewall”, enter the password.
  2. “Create a new port forwarding rule”, give it a name. Under “Select the device …” pick this PC from the device list.
  3. External + Internal port 8787 each, protocol TCP › “Set” › “Save”.
  4. Add an identical second rule for port 8189, protocol UDP.

Sky Router

Black hubs (Sky Hub / Q Hub / Broadband Hub): 192.168.0.1 › “Advanced” tab, sign in.

  1. “LAN IP Setup › Address Reservation › Add”: select this PC, note the IP shown, “Add”.
  2. “Security › Services › Add Service”: Name “Lumora-TCP”, Type “TCP/UDP”, Start + Finish port 8787 › “Apply”. Add a second service “Lumora-UDP” with port 8189/UDP.
  3. “Security › Firewall Rules” › “Inbound Services” › “Add”: pick the service, “Action = ALLOW always”, “Destination LAN address” = the noted PC IP › “Apply”.

White WiFi Max hub (SR213): only via the “My Sky” app under Broadband › Product Settings › Advanced settings.

Virgin Media Hub

Important: the hub must run in Router mode (in Modem mode port forwarding is disabled). Open 192.168.0.1 (settings password on the base).

  1. “Advanced settings › DHCP”: find this PC and note its IP (ideally reserve it).
  2. Open “Advanced settings › Security › Port Forwarding”.
  3. Rule 1: Local + External start/end port 8787 each, protocol TCP, target = the noted PC IP.
  4. Rule 2: ports 8189 each, protocol UDP. Save (a reboot may be needed).

Xfinity Gateway

On Xfinity xFi, port forwarding works only via the Xfinity app (not the web menu).

  1. Open the app › “WiFi › View WiFi equipment › Advanced Settings › Port Forwarding”.
  2. “Add Port Forward” › “Continue”, then select this PC. (Only devices with DHCP + IPv4 appear – turn off the PC's random/private Wi-Fi MAC if needed.)
  3. “Manual Setup”: port 8787, protocol TCP › “Next”.
  4. “Add Port Forward › Manual Setup” again: port 8189, protocol UDP.

If it still won't work despite the rule: in the app under “Advanced Security”, set “Allow Access” for this PC.

AT&T Gateway

AT&T gateways don't offer UPnP – manual forwarding is mandatory. Open 192.168.1.254, enter the “Device Access Code” (on the side/bottom of the device).

  1. Open “Firewall › NAT/Gaming”.
  2. “Custom Services”: Service Name “Lumora-TCP”, Global Port Range 8787 in both fields, Base Host Port 8787, protocol TCP › “Add”.
  3. Add a second custom service “Lumora-UDP” with port 8189, protocol UDP.
  4. “Return to NAT/Gaming” › under “Needed by Device” select this PC › “Add” › “Save”.

No port-forwarding menu? (DS-Lite)

Some connections (many cable and cheap DSL plans) no longer have their own public IPv4 address – this is called DS-Lite. The port-forwarding menu may then be missing entirely, and IPv4 forwarding is technically impossible. Two ways out:

  • Use Lumora's IPv6 direct path – it works for viewers with IPv6 (mobile, modern connections) with no port forwarding at all.
  • Ask your provider for a public IPv4 address / “Dual Stack” (free with many providers).

Frequently asked questions

Why two ports of all things?

Port 8787 (TCP) sets up the connection, port 8189 (UDP) carries the actual video and audio. Both must point to the same PC.

Is this a security risk?

The rule only opens these two ports to exactly your PC, and only Lumora listens on them. The video stream is additionally encrypted (WebRTC/DTLS). When you're not streaming, nothing happens there.

My IP changed and the stream stopped working

Then the PC had no static IP and the rule now points to a different device. Give the PC a static/reserved IP in the router (step 2 of the basic pattern) – then the rule stays valid for good.

Do I really have to do this by hand?

Only if the automatic setup fails. Enable “UPnP” / “automatic port forwarding” in the router and Lumora takes over again on the next stream.
